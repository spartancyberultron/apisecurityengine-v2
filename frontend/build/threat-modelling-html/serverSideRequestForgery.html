Server-Side Request Forgery (SSRF) is a vulnerability that allows an attacker to send crafted requests from a vulnerable web application to internal systems behind the firewall. The attacker can potentially access or manipulate internal resources that were not intended to be exposed. In the context of APIs, this can be particularly concerning because APIs often interact with various internal and external services.

When performing a threat model for SSRF vulnerabilities, particularly API 7:2023, it's important to consider both STRIDE and DREAD models to evaluate the risks comprehensively. Here's how you might categorise the threat based on these models:

STRIDE Model for API 7:2023 (SSRF)

STRIDE is a model used to identify and categorise security threats based on six categories: Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service (DoS), and Elevation of Privilege.

(Critical): SSRF can allow an attacker to impersonate the server or another internal service, making requests that appear legitimate.

Tampering (High): If the SSRF vulnerability allows an attacker to modify internal HTTP requests, this could lead to unauthorised changes in data or system configurations.

Repudiation (Medium): SSRF attacks may not always leave a clear trail, especially if the attacker can mask their actions as legitimate requests from the vulnerable server, complicating auditing and tracking.

Information Disclosure (Critical): SSRF can expose sensitive information from internal services that are not intended to be accessible, posing a significant risk.

Denial of Service (High): By abusing SSRF, an attacker could target internal systems, potentially leading to a denial of service for critical infrastructure.

Elevation of Privilege (High): Depending on the systems accessible via SSRF, an attacker might escalate their privileges within the network.

DREAD Model for API 7:2023 (SSRF)

DREAD is another risk assessment model used to quantify, prioritise, and compare different types of threats. It stands for Damage, Reproducibility, Exploitability, Affected Users, and Discoverability.

Damage (Critical): The potential damage from SSRF can be very high, including data breaches, internal system compromise, and exposure of sensitive information.

Reproducibility (High): If an attacker knows about the SSRF vulnerability, it can be relatively easy to reproduce the attack.

Exploitability (Medium): The difficulty of exploiting SSRF vulnerabilities can vary, but often it requires some knowledge of the internal network and the targeted application's behaviour.

Affected Users (High): Depending on the application and the nature of the SSRF vulnerability, the number of affected users could be extensive, especially if critical systems are compromised.

Discoverability (Medium): Discovering SSRF vulnerabilities might require some effort from the attacker, such as understanding the application's architecture and internal services. However, once identified, the vulnerability can be highly exploitable.
